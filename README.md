# Tutoring Progress App

Веб-приложение для индивидуального репетиторства: публичная страница преподавателя, заявки с сайта, админка, личные кабинеты учеников и родителей, отчеты после уроков, домашние задания, PDF-вложения и прозрачный прогресс по темам.

## Что это

Это MVP личного кабинета для репетитора по математике и информатике. Преподаватель вручную ведет учеников: создает кабинеты, фиксирует цели, текущий уровень, пробелы, темы, отчеты после уроков, домашние задания и чекпоинты. Родитель или ученик входят по 10-значному коду и видят актуальную картину прогресса без регистрации.

Продукт рассчитан на формат `concierge MVP`: максимум полезной прозрачности для родителя и минимум лишней платформенной сложности на старте.

## Быстрый запуск

```bash
cp .env.example .env
docker compose up --build
```

После запуска:

- главная страница: <http://localhost:8000>
- админка преподавателя: <http://localhost:8000/admin>
- вход ученика: <http://localhost:8000/cabinet/login>
- демо-код ученика: `1234567890`

Локальный демо-пароль преподавателя задается через `TEACHER_PASSWORD` в `.env`.

## Переменные окружения

| Переменная | Назначение | Пример |
|---|---|---|
| `DATABASE_URL` | подключение к PostgreSQL | задается в `docker-compose.yml` |
| `PUBLIC_BASE_URL` | базовый URL приложения | `http://localhost:8000` |
| `TEACHER_PASSWORD` | пароль преподавателя | `change-me` |
| `SECRET_KEY` | подпись cookie-сессии | длинная случайная строка |
| `SEED_DEMO_DATA` | создавать демо-данные при пустой БД | `true` |
| `TUTOR_NAME` | имя преподавателя для будущих расширений | `Ваш преподаватель` |
| `TELEGRAM_URL` | ссылка на Telegram | `https://t.me/...` |
| `WHATSAPP_URL` | ссылка на WhatsApp | `https://wa.me/...` |
| `UPLOAD_DIR` | папка PDF-вложений внутри контейнера | `/app/uploads` |

Перед публичной выкладкой обязательно поменяйте `TEACHER_PASSWORD`, `SECRET_KEY` и отключите демо-данные.

## Основные возможности

### Публичная часть

- главная страница с описанием формата занятий;
- отдельная страница `О преподавателе`;
- фото преподавателя;
- блок заявки без регистрации;
- выпадающие списки для класса, предмета и цели;
- вход в личный кабинет через меню:
  - вход для преподавателя;
  - вход для ученика.

### Заявки

Родитель оставляет заявку на сайте. В админке преподаватель видит заявку отдельной карточкой и может:

- посмотреть контактные данные;
- перевести заявку в статус `Новая`, `Связались`, `Одобрено`, `Отклонено`;
- нажать `Связаться`;
- отклонить заявку;
- одобрить заявку и открыть форму создания кабинета с предзаполненными данными.

Статус `Одобрено` ставится только после создания кабинета, а не в момент клика по кнопке.

### Админка преподавателя

В админке есть:

- список учеников;
- карточка каждого ученика;
- создание и редактирование кабинетов;
- темы и статусы прогресса;
- отчеты после уроков;
- домашние задания;
- отдельная страница всех ДЗ: `/admin/homework`;
- материалы и ссылки на доски/конспекты;
- месячные чекпоинты;
- PDF-файлы домашних заданий;
- просмотр PDF-решений, загруженных учеником.

### Личный кабинет ученика

Родитель или ученик открывает `Войти в личный кабинет`, выбирает `Вход для ученика` и вводит 10-значный код, который отправил преподаватель.

В кабинете видны:

- цель;
- текущий статус;
- текущий уровень;
- главные пробелы;
- план на 2-4 недели;
- фокус следующего урока;
- активные домашние задания;
- PDF домашнего задания;
- загрузка PDF-решения;
- статусы тем;
- последний отчет после урока;
- история отчетов;
- материалы;
- чекпоинты.

## Домашние задания и PDF

Преподаватель может:

- создать ДЗ в карточке ученика;
- редактировать ДЗ без удаления;
- прикрепить PDF с заданием;
- отметить ДЗ выполненным;
- открыть PDF-решение ученика;
- видеть все задания на `/admin/homework`.

Ученик или родитель может:

- открыть PDF с заданием;
- прикрепить PDF-решение к конкретному ДЗ.

Решения прикрепляются только со стороны ученика/родителя. У преподавателя нет кнопки загрузки решения вручную.

## Материалы

Материалы в v1 хранятся как ссылки:

- Miro/онлайн-доска;
- Google Docs;
- конспект;
- внешний PDF;
- другой внешний материал.

В кабинете отображается дата и название занятия, чтобы ученик не путался между материалами.

## База данных

Основные таблицы:

- `leads` — заявки с сайта;
- `students` — кабинеты учеников, внутренние access-токены и 10-значные коды входа;
- `topic_progress` — темы и статусы;
- `lesson_reports` — отчеты после уроков;
- `homework` — домашние задания и PDF-метаданные;
- `materials` — ссылки на материалы;
- `checkpoints` — месячные чекпоинты.

Схема создается автоматически через SQLAlchemy. Для локальных MVP-изменений есть легкие startup-миграции в `app/migrations.py`.

## Типовой сценарий

1. Родитель оставляет заявку на главной странице.
2. Преподаватель открывает `/admin`.
3. Преподаватель связывается с родителем и меняет статус заявки.
4. Если заявка подходит, преподаватель нажимает `Одобрить`.
5. Открывается создание кабинета с предзаполненными данными.
6. Преподаватель заполняет цель, уровень, пробелы и план.
7. После `Создать кабинет` заявка становится `Одобрено`.
8. Преподаватель копирует 10-значный код и отправляет родителю.
9. Родитель или ученик входит по коду.
10. После уроков преподаватель добавляет отчеты, ДЗ, материалы и чекпоинты.
11. Ученик загружает PDF-решения к конкретным заданиям.

## Ограничения v1

Пока намеренно не реализованы:

- саморегистрация родителей;
- отдельные аккаунты учеников;
- онлайн-оплаты;
- расписание и календарь;
- встроенный чат;
- уведомления;
- несколько преподавателей;
- сложная аналитика и графики.

Эти функции стоит добавлять после проверки, что родители действительно ценят прозрачный прогресс, понятные отчеты и кодовый доступ к кабинету.

## Безопасность перед продакшеном

Перед публичным использованием:

- замените `TEACHER_PASSWORD`;
- замените `SECRET_KEY`;
- отключите `SEED_DEMO_DATA`;
- включите HTTPS;
- замените локальные пароли PostgreSQL;
- настройте backup базы;
- храните PDF-вложения в приватном storage;
- добавьте CSRF-защиту для форм;
- продумайте политику удаления персональных данных.
