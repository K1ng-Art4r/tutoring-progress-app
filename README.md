# Tutoring Progress App

Веб-приложение для индивидуального репетиторства по математике: публичная страница преподавателя, заявки с сайта, админка, личные кабинеты учеников и родителей, диагностические работы, отчеты после уроков, домашние задания, PDF-вложения и прозрачный прогресс по темам.

## Что это

Это MVP личного кабинета для репетитора по математике: ОГЭ, ЕГЭ база и ЕГЭ профиль. Преподаватель вручную ведет учеников: создает кабинеты, фиксирует цели, текущий уровень, пробелы, темы, диагностические работы, отчеты после уроков, домашние задания и чекпоинты. Родитель или ученик входят по 10-значному коду и видят актуальную картину прогресса без регистрации.

Продукт рассчитан на формат `concierge MVP`: максимум полезной прозрачности для родителя и минимум лишней платформенной сложности на старте.

## Быстрый запуск

```bash
cp .env.example .env
docker compose up --build
```

После запуска:

- главная страница: <http://localhost:8000>
- админка преподавателя: <http://localhost:8000/admin>
- вход ученика: <http://localhost:8000/cabinet/login>
- демо-код ученика: `1234567890`

Локальный демо-пароль преподавателя задается через `TEACHER_PASSWORD` в `.env`.

## Переменные окружения

| Переменная | Назначение | Пример |
|---|---|---|
| `DATABASE_URL` | подключение к PostgreSQL | задается в `docker-compose.yml` |
| `PUBLIC_BASE_URL` | базовый URL приложения | `http://localhost:8000` |
| `TEACHER_PASSWORD` | пароль преподавателя | `change-me` |
| `SECRET_KEY` | подпись cookie-сессии | длинная случайная строка |
| `SEED_DEMO_DATA` | создавать демо-данные при пустой БД | `true` |
| `TUTOR_NAME` | имя преподавателя для будущих расширений | `Ваш преподаватель` |
| `TELEGRAM_URL` | ссылка на Telegram | `https://t.me/...` |
| `WHATSAPP_URL` | ссылка на WhatsApp | `https://wa.me/...` |
| `UPLOAD_DIR` | папка PDF-вложений внутри контейнера | `/app/uploads` |

Перед публичной выкладкой обязательно поменяйте `TEACHER_PASSWORD`, `SECRET_KEY` и отключите демо-данные.

## Публикация на Railway

Проект готов к деплою на Railway через GitHub. Railway будет собирать приложение по `Dockerfile`, читать настройки из `railway.toml`, подключать PostgreSQL через `DATABASE_URL` и проверять работоспособность по `/healthz`.

### 1. Подготовить репозиторий

Убедитесь, что все нужные изменения закоммичены и отправлены в GitHub:

```bash
git status
git add .
git commit -m "Prepare Railway deployment"
git push origin main
```

### 2. Создать проект в Railway

1. Откройте [railway.com](https://railway.com).
2. Создайте новый проект из GitHub-репозитория `K1ng-Art4r/tutoring-progress-app`.
3. Выберите ветку `main`.
4. Добавьте сервис PostgreSQL через `+ New` → `Database` → `PostgreSQL`.

Railway автоматически даст PostgreSQL-переменные, включая `DATABASE_URL`. Приложение умеет принимать Railway-формат `postgresql://...` и само переводит его на драйвер `psycopg`.

### 3. Переменные web-сервиса

В сервисе приложения в Railway задайте:

| Переменная | Значение |
|---|---|
| `DATABASE_URL` | ссылка на `DATABASE_URL` PostgreSQL-сервиса |
| `TEACHER_PASSWORD` | новый пароль преподавателя |
| `SECRET_KEY` | длинная случайная строка |
| `SEED_DEMO_DATA` | `false` |
| `TUTOR_NAME` | `Давыдов Евгений Алексеевич` |
| `TELEGRAM_URL` | ссылка на Telegram |
| `WHATSAPP_URL` | ссылка на WhatsApp |
| `UPLOAD_DIR` | `/app/uploads` |

`PUBLIC_BASE_URL` можно не задавать, если используете домен Railway: приложение возьмет `RAILWAY_PUBLIC_DOMAIN`. Если подключаете свой домен, задайте `PUBLIC_BASE_URL=https://ваш-домен`.

### 4. Сохранение PDF и решений

В приложении есть загрузка PDF-домашек и файлов решений диагностики. Для публичного режима нужен постоянный диск:

1. В Railway откройте web-сервис приложения.
2. Добавьте Volume.
3. Укажите mount path: `/app/uploads`.
4. Оставьте `UPLOAD_DIR=/app/uploads`.

Без Volume файлы могут пропасть после пересборки или перезапуска контейнера.

### 5. Открыть сайт наружу

В Railway откройте `Settings` → `Networking` → `Public Networking` → `Generate Domain`. После этого сайт будет доступен по HTTPS-домену Railway. Позже можно подключить свой домен.

### 6. Как редактировать после публикации

Базовый сценарий:

```bash
git pull
# вносите изменения
docker compose up --build
git add .
git commit -m "Update site"
git push origin main
```

После `git push` Railway автоматически пересоберет и выкатит новую версию. Данные учеников, заявок, диагностик и домашних заданий останутся в PostgreSQL, а загруженные файлы останутся на Volume.

## Основные возможности

### Публичная часть

- главная страница с описанием формата занятий;
- отдельная страница `О преподавателе`;
- отдельная страница `Цены`: `/pricing`;
- фото преподавателя;
- блок заявки без регистрации;
- выпадающие списки для класса, направления математики и цели;
- вход в личный кабинет через меню:
  - вход для преподавателя;
  - вход для ученика.

### Заявки

Родитель оставляет заявку на сайте. В админке преподаватель видит заявку отдельной карточкой и может:

- посмотреть контактные данные;
- перевести заявку в статус `Новая`, `Связались`, `Одобрено`, `Отклонено`;
- нажать `Связаться`;
- отклонить заявку;
- одобрить заявку и открыть форму создания кабинета с предзаполненными данными.

Статус `Одобрено` ставится только после создания кабинета, а не в момент клика по кнопке.

### Админка преподавателя

В админке есть:

- список учеников;
- карточка каждого ученика;
- создание и редактирование кабинетов;
- темы и статусы прогресса;
- отчеты после уроков;
- домашние задания;
- отдельная страница всех ДЗ: `/admin/homework`;
- отдельная страница диагностик: `/admin/diagnostics`;
- проверка диагностических работ с автосверкой ответов;
- ручная корректировка баллов и комментариев по заданиям;
- автоматический вывод уровня и текст для родителя после проверки;
- материалы и ссылки на доски/конспекты;
- месячные чекпоинты;
- PDF-файлы домашних заданий;
- просмотр PDF-решений, загруженных учеником.

### Личный кабинет ученика

Родитель или ученик открывает `Войти в личный кабинет`, выбирает `Вход для ученика` и вводит 10-значный код, который отправил преподаватель.

В кабинете видны:

- цель;
- текущий статус;
- текущий уровень;
- главные пробелы;
- план на 2-4 недели;
- фокус следующего урока;
- активные домашние задания;
- PDF домашнего задания;
- загрузка PDF-решения;
- доступные диагностические работы по своему направлению;
- таймер диагностики на 40 минут;
- сохранение ответа по каждому заданию;
- загрузка скриншота или PDF решения для заданий с развернутым ответом;
- результат диагностики после проверки преподавателем;
- разбор проверенной диагностики: задания, ответы ученика, правильные ответы, баллы и краткие решения;
- статусы тем;
- последний отчет после урока;
- история отчетов;
- материалы;
- чекпоинты.

## Домашние задания и PDF

Преподаватель может:

- создать ДЗ в карточке ученика;
- редактировать ДЗ без удаления;
- прикрепить PDF с заданием;
- отметить ДЗ выполненным;
- открыть PDF-решение ученика;
- видеть все задания на `/admin/homework`.

Ученик или родитель может:

- открыть PDF с заданием;
- прикрепить PDF-решение к конкретному ДЗ.

Решения прикрепляются только со стороны ученика/родителя. У преподавателя нет кнопки загрузки решения вручную.

## Диагностические работы

В текущей версии добавлены диагностические работы для направлений:

- `Математика, ЕГЭ профиль`;
- `Математика, ЕГЭ база`;
- `Математика, ОГЭ`.

В каждой диагностике:

- 12 заданий;
- 40 минут на выполнение;
- максимум 20 диагностических баллов;
- задания 1-10 с кратким ответом;
- задания 11-12 требуют ответа и прикрепленного решения;
- автосверка ответа с правильным ответом;
- ручная перепроверка преподавателем;
- отдельная шкала уровня для конкретного экзамена;
- автоматический вывод и готовый текст для родителя.

Ученик открывает работу в личном кабинете. На отдельной странице есть таймер, навигация по заданиям и кнопка `Сохранить ответ`. Когда время заканчивается или ученик нажимает `Завершить диагностику`, работа становится недоступна для изменения и появляется в админке в разделе `Диагностики`.

Преподаватель открывает `/admin/diagnostics`, выбирает работу со статусом `Нужна проверка`, видит:

- условие каждого задания;
- ответ ученика;
- правильный ответ;
- автоматический балл;
- прикрепленное решение;
- критерии;
- поле ручного балла;
- комментарий по заданию.

После сохранения проверки кабинет ученика показывает итоговый балл, вывод, текст для родителя и полный разбор: условие каждого задания, ответ ученика, правильный ответ, комментарий преподавателя и краткое решение.

Для `Математика, ЕГЭ база` добавлена отдельная диагностика с графиком в задании 8, шкалой 0-6 / 7-10 / 11-15 / 16-20 и выводами под базовый уровень.

## Цены

На странице `/pricing` показаны два формата:

- `Почасовая оплата` — `2 500 ₽` за 60 минут.
- `Блок на 4 недели` — `20 000 ₽` за 8 занятий, стартовую диагностику, план подготовки и дополнительную диагностику в конце периода.

Блок сделан как рекомендуемый формат для подготовки к ОГЭ и ЕГЭ, потому что там есть регулярность, стартовый срез и финальная проверка результата.

## Материалы

Материалы в v1 хранятся как ссылки:

- Miro/онлайн-доска;
- Google Docs;
- конспект;
- внешний PDF;
- другой внешний материал.

В кабинете отображается дата и название занятия, чтобы ученик не путался между материалами.

## База данных

Основные таблицы:

- `leads` — заявки с сайта;
- `students` — кабинеты учеников, внутренние access-токены и 10-значные коды входа;
- `topic_progress` — темы и статусы;
- `lesson_reports` — отчеты после уроков;
- `homework` — домашние задания и PDF-метаданные;
- `materials` — ссылки на материалы;
- `checkpoints` — месячные чекпоинты;
- `diagnostic_works` — диагностические работы;
- `diagnostic_tasks` — задания диагностик;
- `diagnostic_attempts` — попытки учеников;
- `diagnostic_answers` — ответы и прикрепленные решения.

Схема создается автоматически через SQLAlchemy. Для локальных MVP-изменений есть легкие startup-миграции в `app/migrations.py`.

## Типовой сценарий

1. Родитель оставляет заявку на главной странице.
2. Преподаватель открывает `/admin`.
3. Преподаватель связывается с родителем и меняет статус заявки.
4. Если заявка подходит, преподаватель нажимает `Одобрить`.
5. Открывается создание кабинета с предзаполненными данными.
6. Преподаватель заполняет цель, уровень, пробелы и план.
7. После `Создать кабинет` заявка становится `Одобрено`.
8. Преподаватель копирует 10-значный код и отправляет родителю.
9. Родитель или ученик входит по коду.
10. Ученик проходит доступную диагностическую работу.
11. Преподаватель проверяет диагностику и получает готовый вывод для родителя.
12. Ученик открывает диагностику и видит разбор с правильными ответами и решениями.
13. После уроков преподаватель добавляет отчеты, ДЗ, материалы и чекпоинты.
14. Ученик загружает PDF-решения к конкретным заданиям.

## Ограничения v1

Пока намеренно не реализованы:

- саморегистрация родителей;
- отдельные аккаунты учеников;
- онлайн-оплаты;
- расписание и календарь;
- встроенный чат;
- уведомления;
- несколько преподавателей;
- сложная аналитика и графики.

Эти функции стоит добавлять после проверки, что родители действительно ценят прозрачный прогресс, понятные отчеты и кодовый доступ к кабинету.

## Безопасность перед продакшеном

Перед публичным использованием:

- замените `TEACHER_PASSWORD`;
- замените `SECRET_KEY`;
- отключите `SEED_DEMO_DATA`;
- включите HTTPS;
- замените локальные пароли PostgreSQL;
- настройте backup базы;
- храните PDF-вложения в приватном storage;
- добавьте CSRF-защиту для форм;
- продумайте политику удаления персональных данных.
